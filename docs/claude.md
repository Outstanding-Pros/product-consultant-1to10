## 프로덕트 개요

**CoPO**은 DAU는 있지만 수익화에 어려움을 겪는 인디 앱 개발자를 위한 맞춤형 수익화 컨설팅 서비스입니다.

### 핵심 컨셉

- **처방전(Prescription)**: 의사가 환자의 증상을 진단하고 맞춤 처방을 내리듯, 앱의 현재 상태를 진단하고 구체적인 수익화 전략을 제시
- **데이터 기반**: 일반론이 아닌, 실제 앱의 DAU/MAU, 카테고리, 현재 수익 모델을 기반으로 한 맞춤 분석
- **실행 가능성**: "이론은 알겠는데..."가 아닌, "이번 주에 이걸 하세요" 수준의 구체적 액션 아이템

### 배경

많은 인디 앱 개발자들이 겪는 공통적인 문제:

1. **광고 수익 천장**: 인앱 광고만으로는 월 100만원 수익을 달성하기 위해 현실적으로 감당하기 어려운 수준의 트래픽이 필요
2. **인앱결제 진입장벽**: 기술이 아니라 "무엇을 어떤 가격에 팔 것인가"의 문제
3. **의사결정 마비**: 앱이 여러 개인 경우, 어디에 집중할지 판단 프레임 부재
4. **고립된 학습**: 강의는 범용적이고, 커뮤니티는 일회성이며, 맥락이 끊김

## 타겟 사용자

### Primary Persona: "광고 수익 정체형 개발자"

**배경:**
- DAU 1,000~5,000 수준의 앱을 운영 중
- 인앱 광고로 월 20~60만원 수익을 내고 있음
- 인앱결제/구독 도입의 필요성은 인지하지만, 무엇을 팔지 결정하지 못한 상태
- 유튜브 강의, 블로그 글은 많이 봤지만 "내 앱"에 직접 적용하기 어려움

**공통 고민:**
- "어떤 기능을 프리미엄으로 만들어야 할까?"
- "구독 가격을 얼마로 설정해야 할까?"
- "과연 내 앱 유저가 돈을 낼까?"

**목표:**
- 광고 외 수익원 확보
- 월 수익 100만원 이상 달성
- 구독 모델 안정화

### Secondary Persona: "다작형 개발자"

**배경:**
- 앱 2~5개를 동시 운영
- 각 앱의 수익이 모두 중간 수준 (월 10~50만원)
- 시간과 리소스가 분산되어 집중력 부족

**공통 고민:**
- "어떤 앱에 집중해야 할까?"
- "어떤 앱은 유지하고, 어떤 앱은 포기해야 할까?"
- "포트폴리오 전체를 어떻게 최적화할까?"

**목표:**
- 데이터 기반 의사결정
- 리소스 집중으로 효율성 향상
- 포트폴리오 수익 극대화

## 핵심 가치 제안

### 1. 맞춤형 (Customized)

**❌ 강의/유튜브:**
- "일반적으로 구독 모델이 좋습니다"
- "대부분의 앱은..."
- 범용 프레임워크

**✅ CoPO:**
- "당신의 앱은 MAU 12,000에 유틸리티 카테고리이므로..."
- "예상 전환율 2~4%로 계산하면..."
- 내 앱 데이터 기반 분석

### 2. 연속성 (Continuity)

**❌ 커뮤니티:**
- 매번 상황 설명부터 시작
- 이전 대화 기억 안 함
- 일회성 질답

**✅ CoPO:**
- 월간 구독 시 이전 실험 결과 기억
- "지난달 A/B 테스트 결과를 보니..."
- 맥락이 끊기지 않는 전략 파트너

### 3. 실행력 (Actionable)

**❌ 일반 컨설팅:**
- "유저 니즈를 파악하세요"
- "시장 조사를 해보세요"
- 추상적인 조언

**✅ CoPO:**
- "이번 주에 '광고 제거 + 알림' 번들을 2,900원에 테스트하세요"
- "Week 1~4: 프리토타입 → Week 5~8: 가격 최적화"
- 구체적인 액션 아이템

### 4. 투명성 (Transparent)

**❌ 추상적 예측:**
- "수익이 증가할 것입니다"
- "많은 효과를 볼 수 있습니다"
- 근거 없는 장밋빛 전망

**✅ CoPO:**
- "MAU 12,000 × 전환율 2~4% = 유료 구독자 240~480명"
- "240~480명 × 월 2,900원 = 월 69.6~139.2만원"
- 계산 근거와 가정을 투명하게 공개

## 결제 시스템

### 결제 플랫폼: Polar.sh

> ⚠️ Toss Payments는 현재 미사용. Polar.sh 단일 결제로 운영.

### Polar.sh

**선택 이유:**
- 인디 개발자 친화적
- 구독 관리 자동화
- 글로벌 카드 결제

**특장점:**
- Stripe 기반으로 안정성 높음
- 대시보드에서 구독 관리 용이
- Webhook으로 자동화 가능
- 결제 링크 공유 가능

**사용자 경험:**
1. 폼 작성 후 결제 버튼 클릭
2. Polar.sh Checkout 페이지로 리다이렉트
3. 카드 정보 입력
4. 결제 완료 후 `/success` 페이지로 리턴

**기술 스택:**
- `@polar-sh/sdk` 사용
- Checkout Session 생성
- 메타데이터로 폼 데이터 전달

### UI/UX 설계

**결제 플로우:** 폼 작성 → Polar.sh Checkout 리다이렉트 → 결제 완료

### 데이터 수집

**결제 시 수집 정보:**
- 앱 이름
- 플랫폼 (Android/iOS/웹/크로스플랫폼)
- 카테고리 (유틸리티, 게임, 라이프스타일 등)
- DAU (일 평균 활성 유저)
- MAU (월 평균 활성 유저)
- 현재 수익 모델 (인앱 광고, 인앱결제, 구독 등)
- 월 수익
- 운영 중인 앱 개수
- 목표 월 수익
- 가장 고민되는 것 (자유 텍스트)
- 이메일 (진단서 전달용)

**데이터 저장:**
- Polar.sh: Checkout Session metadata

## 운영자 소개

### 배경 및 경력

**이름:** [운영자 이름]

**타이틀:**
- **前 프로덕트 오너(PO)** ← 핵심 공신력 포인트
- 제품 빌더 (CoPO 설계/운영)
- PO/GTM 네트워크 운영자 (250명 규모)

### PO 경력 강조

**왜 PO 출신이 중요한가?**

1. **데이터 기반 의사결정 능력**
   - PO로서 A/B 테스트, 지표 분석, 가설 검증 경험
   - 감이 아닌 데이터로 전략 수립

2. **사용자 중심 사고**
   - 유저 페르소나, 사용자 여정 설계 경험
   - "무엇을 팔 것인가"를 사용자 니즈 기반으로 판단

3. **제품 전략 수립 경험**
   - 로드맵 설계, 우선순위 결정, 리소스 배분
   - 앱 개발자의 고민을 제품 관점에서 이해

4. **실무 경험**
   - 이론만 아는 컨설턴트가 아님
   - 실제 제품을 만들고, 성장시킨 경험

### 스토리텔링

**"PO 실무를 제품으로 만들었습니다"**

```
PO로 일하며 반복했던 A/B 테스트, 지표 분석, 가설 검증 프로세스를
인디 앱 수익화에 맞게 제품으로 구조화했습니다.

사람이 매번 감으로 판단하는 컨설팅이 아니라,
프레임워크가 일관되게 적용되는 제품으로 만들었습니다.

PO/GTM 네트워크 250명 규모에서 축적되는 현장 케이스를 바탕으로
광고 단일 의존에서 구독/인앱결제 전환까지의 패턴을
지속적으로 업데이트하고 있습니다.
```

### 신뢰 요소

**자격/경험:**
- 前 프로덕트 오너 (PO) - 제품 전략 전문가
- PO/GTM 네트워크 운영 (250명)
- 제품화된 수익화 프레임워크 설계/운영
- AI 기반 전략 분석 프레임워크 개발

**차별점:**
- 사람 역량 의존 컨설팅이 아닌 제품화된 워크플로우
- PO 실행 방식을 반영한 일관된 데이터 기반 분석
- 네트워크 케이스 반영으로 프레임워크를 지속 고도화

### 진단 프레임워크 (일부 공개)

**4단계 분석:**

1. **수익 구조 진단**
   - 현재 ARPU (Average Revenue Per User)
   - 수익 모델별 효율 분석
   - 수익 모델별 잠재력 평가

2. **전환 가능성 평가**
   - 카테고리별 벤치마크 대비 비교
   - 구독/인앱 전환율 추정
   - 유저 세그먼트 분석

3. **가격 시뮬레이션**
   - WTP (Willingness To Pay) 기반 최적 가격대
   - 수익 극대화 포인트 산출
   - 시나리오별 예상 수익

4. **실행 로드맵**
   - 30/60/90일 단위 액션 플랜
   - 우선순위 기반 순서 설계
   - 리스크 최소화 전략

## 주요 기능

### 1. 진단서 생성

**입력:**
- 앱 프로필 (이름, 카테고리, DAU/MAU 등)
- 현재 수익 모델 및 월 수익
- 주요 고민 사항

**처리:**
- AI 기반 1차 분석
- 카테고리별 벤치마크 데이터 매칭
- 제품화된 PO 프레임워크 적용 및 점수화

**출력:**
- 5단계 수익화 진단
- 추천 수익 모델
- 예상 수익 시뮬레이션 (계산 근거 포함)
- 30/60/90일 실행 로드맵

### 2. 월간 구독 (연속 코칭)

**특징:**
- 이전 실험 결과 기억
- "Week 1에 A 실험 → Week 5에 결과 보고 → Week 6에 B 실험" 연속성
- 맥락이 끊기지 않는 전략 파트너
